---
title: "Quickstart: Generate, Serve, and Benchmark with DFlash"
description: "Go from install to your first DFlash generation, a running OpenAI-compatible server, a connected client, and a benchmark result in five steps."
---

This quickstart walks through the full dflash-mlx workflow: generating text from the CLI, starting an OpenAI-compatible server, querying it with curl, and measuring throughput with the benchmark tool. All commands use real examples from the library.

<Tip>
  Chat templates are enabled by default in both `dflash` and `dflash-serve`. The prompt you pass is automatically wrapped in the model's chat template before tokenization. Pass `--no-chat-template` to the CLI or `--chat-template-args '{"enable_thinking": false}'` to the server to adjust template behavior.
</Tip>

<Steps>
  <Step title="Install dflash-mlx">
    If you haven't installed yet, follow the [installation guide](/installation). The short version:

    ```bash
    pip install dflash-mlx
    ```

    Verify the CLI is available:

    ```bash
    dflash --help
    ```
  </Step>

  <Step title="Generate text with the dflash CLI">
    Run a generation against a supported target model. The draft model is resolved automatically — no `--draft` flag needed.

    ```bash
    PROMPT='The function $f$ satisfies the functional equation \[ f(x) + f(y) = f(x + y) - xy - 1 \] for all real numbers $x$ and $y$. If $f(1) = 1$, then find all integers $n$ such that $f(n) = n$. Enter all such integers, separated by commas. Please reason step by step, and put your final answer within \boxed{}.'

    dflash --model Qwen/Qwen3.5-9B --prompt "$PROMPT"
    ```

    Tokens stream to stdout as they are committed. A summary line is printed to stderr when generation completes:

    ```
    1024 tokens | 127.1 tok/s | 89.2% acceptance
    ```

    To use a specific draft model instead of the auto-resolved one:

    ```bash
    dflash --model Qwen/Qwen3.5-9B \
           --draft z-lab/Qwen3.5-9B-DFlash \
           --prompt "$PROMPT"
    ```

    <Note>
      Auto draft resolution matches the model name against the built-in registry. Supported base models include `Qwen3.5-4B`, `Qwen3.5-9B`, `Qwen3.5-27B`, and `Qwen3.5-35B-A3B`. Pass `--draft` explicitly for any model not in the registry.
    </Note>
  </Step>

  <Step title="Start an OpenAI-compatible server">
    `dflash-serve` wraps `mlx_lm.server` with the DFlash runtime as the generation engine. It supports streaming SSE, tool calling, and reasoning — compatible with Open WebUI, Continue, aider, OpenCode, and any other OpenAI client.

    ```bash
    dflash-serve --model Qwen/Qwen3.5-9B --port 8000
    ```

    To disable visible thinking/reasoning tokens on models that support it:

    ```bash
    dflash-serve --model Qwen/Qwen3.5-9B --port 8000 \
      --chat-template-args '{"enable_thinking": false}'
    ```

    For larger MoE models, raise the DFlash context threshold to extend speculative decoding to longer prompts:

    ```bash
    dflash-serve --model mlx-community/Qwen3.5-35B-A3B-4bit --port 8000 \
      --chat-template-args '{"enable_thinking": false}' \
      --dflash-max-ctx 16384
    ```

    The server starts on the specified port and logs each request to stderr. Leave it running and proceed to the next step.
  </Step>

  <Step title="Connect a client">
    With the server running on port 8000, send a request using curl or any OpenAI-compatible client:

    ```bash
    curl http://localhost:8000/v1/chat/completions \
      -H "Content-Type: application/json" \
      -d '{
        "model": "Qwen/Qwen3.5-9B",
        "messages": [
          {
            "role": "user",
            "content": "Explain the difference between speculative decoding and standard autoregressive generation."
          }
        ],
        "stream": false
      }'
    ```

    For streaming responses, set `"stream": true` and the server will emit Server-Sent Events as tokens are committed by the DFlash runtime.

    The server is also compatible with the OpenAI Python SDK:

    ```python
    from openai import OpenAI

    client = OpenAI(base_url="http://localhost:8000/v1", api_key="local")

    response = client.chat.completions.create(
        model="Qwen/Qwen3.5-9B",
        messages=[{"role": "user", "content": "What is speculative decoding?"}],
    )
    print(response.choices[0].message.content)
    ```
  </Step>

  <Step title="Run a benchmark">
    Use `dflash-benchmark` to measure DFlash throughput against the baseline autoregressive speed. Results include per-run token counts, tokens per second, and acceptance rates.

    ```bash
    PROMPT='The function $f$ satisfies the functional equation \[ f(x) + f(y) = f(x + y) - xy - 1 \] for all real numbers $x$ and $y$. If $f(1) = 1$, then find all integers $n$ such that $f(n) = n$. Enter all such integers, separated by commas. Please reason step by step, and put your final answer within \boxed{}.'

    dflash-benchmark \
      --model Qwen/Qwen3.5-9B \
      --draft z-lab/Qwen3.5-9B-DFlash \
      --prompt "$PROMPT" \
      --max-tokens 1024 \
      --repeat 3 \
      --no-eos
    ```

    The benchmark runs both baseline (`mlx_lm.stream_generate`) and DFlash sequentially on the same model, with a cooldown between runs, and reports the median result. Full per-run JSON reports are written to `benchmark/results/`.

    Expected output for Qwen3.5-9B at 1024 tokens on an M5 Max:

    ```
    baseline:  31.09 tok/s
    dflash:   127.47 tok/s  (4.10x)  88.96% acceptance
    ```
  </Step>
</Steps>
