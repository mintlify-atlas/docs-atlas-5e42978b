When you run `dflash` or `dflash-serve` without a `--draft` flag, the runtime automatically selects a compatible DFlash draft model by looking up your target model reference in a built-in registry. This means most users never need to specify a draft manually — passing just `--model` is enough for any model that has a registry entry.

## DRAFT\_REGISTRY

The registry is defined in `dflash_mlx/generate.py` as a plain Python dict mapping model name suffixes to HuggingFace draft repo IDs:

```python
DRAFT_REGISTRY = {
    "Qwen3.5-4B": "z-lab/Qwen3.5-4B-DFlash",
    "Qwen3.5-9B": "z-lab/Qwen3.5-9B-DFlash",
    "Qwen3.5-27B": "z-lab/Qwen3.5-27B-DFlash",
    "Qwen3.5-35B-A3B": "z-lab/Qwen3.5-35B-A3B-DFlash",
    "Qwen3-4B": "z-lab/Qwen3-4B-DFlash-b16",
    "Qwen3-8B": "z-lab/Qwen3-8B-DFlash-b16",
}
```

Keys are the base model name without an org prefix. Values are full HuggingFace repo IDs. The registry is normalized to lowercase internally before matching, so lookups are case-insensitive.

## How resolution works

Resolution is handled by `resolve_optional_draft_ref()`. The function takes the full target model reference (e.g. `Qwen/Qwen3.5-9B`) and an optional explicit draft override.

```python
def resolve_optional_draft_ref(model_ref: str, draft_ref: Optional[str]) -> Optional[str]:
    if draft_ref:
        return draft_ref

    stripped_name = _strip_model_org(model_ref)   # "Qwen3.5-9B"
    lowered_name = stripped_name.lower()           # "qwen3.5-9b"

    exact = _NORMALIZED_DRAFT_REGISTRY.get(lowered_name)
    if exact is not None:
        return exact

    matching_bases = [
        base_name
        for base_name in _NORMALIZED_DRAFT_REGISTRY
        if lowered_name == base_name
        or lowered_name.startswith(base_name + "-")
        or lowered_name.startswith(base_name + "_")
    ]
    if not matching_bases:
        return None

    best_match = max(matching_bases, key=len)
    return _NORMALIZED_DRAFT_REGISTRY[best_match]
```

The resolution steps are:

1. **Explicit override** — if `draft_ref` is provided, it is returned immediately without consulting the registry.
2. **Strip org prefix** — the function calls `_strip_model_org()`, which takes everything after the last `/` in the model reference. `Qwen/Qwen3.5-9B` becomes `Qwen3.5-9B`.
3. **Exact match** — the stripped name is lowercased and looked up directly in the normalized registry.
4. **Prefix match** — if no exact match, all registry keys that are a prefix of the lowercased name (followed by `-` or `_`) are collected. The longest matching key wins. This handles quantized or fine-tuned variants like `Qwen3.5-9B-Instruct` resolving to the same draft as `Qwen3.5-9B`.
5. **No match** — if no key matches, `None` is returned. The caller (`load_runtime_components`) then raises a `ValueError`.

## Overriding the registry

Pass `--draft` to bypass the registry entirely and use any HuggingFace repo or local path as the draft model:

```bash
# CLI
dflash --model Qwen/Qwen3.5-9B \
       --draft z-lab/Qwen3.5-9B-DFlash \
       --prompt "Your prompt"

# Server
dflash-serve --model Qwen/Qwen3.5-9B \
             --draft z-lab/Qwen3.5-9B-DFlash \
             --port 8000
```

When `--draft` is set, `resolve_optional_draft_ref()` returns it immediately at step 1 and the registry is never consulted.

## Python API

`resolve_optional_draft_ref` is importable directly if you are building on top of the dflash-mlx runtime:

```python
from dflash_mlx.generate import resolve_optional_draft_ref

# Auto-resolve from registry
draft = resolve_optional_draft_ref("Qwen/Qwen3.5-9B", None)
# → "z-lab/Qwen3.5-9B-DFlash"

# Explicit override — registry is bypassed
draft = resolve_optional_draft_ref("Qwen/Qwen3.5-9B", "my-org/my-draft")
# → "my-org/my-draft"

# No registry entry — returns None
draft = resolve_optional_draft_ref("meta-llama/Llama-3-8B", None)
# → None
```

**Signature:**

```python
def resolve_optional_draft_ref(
    model_ref: str,
    draft_ref: Optional[str],
) -> Optional[str]: ...
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `model_ref` | `str` | Full target model reference (HuggingFace repo ID or local path). |
| `draft_ref` | `Optional[str]` | Explicit draft override. Pass `None` to use the registry. |
| **Returns** | `Optional[str]` | Resolved draft repo ID, or `None` if no match found. |

<Note>
  Draft models are downloaded from HuggingFace on first use and cached locally by `huggingface_hub`. Subsequent runs load from the local cache and do not require a network connection. Use `HF_HOME` or `HUGGINGFACE_HUB_CACHE` environment variables to control the cache location.
</Note>
